## ゲーム概要

LoL（League of Legends）をベースにしたGMレスTRPG。
ダイスロールとマッチアップ判定を用いて、戦略的な5vs5チーム戦を再現する。

### 基本コンセプト
- **GMレス**: ゲームマスター不要で自動進行
- **ダイスベース**: ランダム性と戦略性のバランス
- **5vs5チーム戦**: ロール分担による協力プレイ
- **戦略性重視**: 視界管理、マッチアップ、リソース配分

## 基本ルール

### チーム構成
- **プレイヤー数**: 5vs5（計10人）
- **チーム**: ブルーチーム vs レッドチーム

### レーンとロールの区別

**重要**: このゲームでは「レーン」と「ロール」を明確に区別します。

#### レーン（配置位置）
各プレイヤーは以下のいずれかのレーンを担当：

1. **TOP（トップレーナー）**
   - 担当レーン: TOPレーン
   - 特徴: 1v1の戦闘力重視

2. **JG（ジャングラー）**
   - 担当: 森エリア
   - 特徴: 機動力、ガンク能力
   - 特殊能力: バフファーム、ガンク時の視界ボーナス

3. **MID（ミッドレーナー）**
   - 担当レーン: MIDレーン
   - 特徴: 高いバースト火力、ローム能力

4. **BOT（ボットレーナー）**
   - 担当レーン: BOTレーン
   - 特徴: 通常2人が配置（ADC + Support など）

**注意**: レーンは単なる配置位置です。どのロールでもどのレーンに配置可能です。

#### ロール（キャラクター特性・ステータス）
プレイヤーキャラクターは以下のロール分類に基づいてステータスが決定されます：

**ロール6種類**:
- **Tank（タンク）**: 高耐久、低火力
- **Fighter（ファイター）**: バランス型、継続戦闘
- **Assassin（アサシン）**: 高機動、高リスク高リターン
- **Mage（メイジ）**: 魔法火力特化
- **Marksman（マークスマン）**: 物理火力特化、後半強化
- **Support（サポート）**: 支援特化

**ダメージタイプ2種類**:
- **AD（物理ダメージ）**: Attack Damage
- **AP（魔法ダメージ）**: Ability Power

詳細なステータスは「キャラクターシステム」ドキュメントを参照してください。

## マップ構造

### タワー配置（各チーム）
各チームは以下の構造を持つ：

**ブルーチーム（左下）/ レッドチーム（右上）**

マップは3つのレーンで構成され、レーン間にジャングル（森）が配置される：

```
                        [敵ネクサス]
                             |
                      [敵ネクサスタワー]
                             |
              +--------------+--------------+
              |              |              |
         [TOPインナー]  [MIDインナー]  [BOTインナー]
              |              |              |
         [TOPアウター]       |        [BOTアウター]
              |              |              |
              |         [MIDアウター]       |
              |              |              |
        +-----+              |              +-----+
        |  敵側TOP-MID森     |      敵側MID-BOT森  |
        |  (赤バフ/青バフ)   |     (赤バフ/青バフ)  |
        +-----+              |              +-----+
              |              |              |
              +------[中立エリア]-------+
                     (ドラゴン/バロン)
              |              |              |
        +-----+              |              +-----+
        | 味方TOP-MID森      |      味方MID-BOT森   |
        | (赤バフ/青バフ)    |     (赤バフ/青バフ)  |
        +-----+              |              +-----+
              |              |              |
              |         [MIDアウター]       |
              |              |              |
         [TOPアウター]       |        [BOTアウター]
              |              |              |
         [TOPインナー]  [MIDインナー]  [BOTインナー]
              |              |              |
              +--------------+--------------+
                             |
                     [味方ネクサスタワー]
                             |
                        [味方ネクサス]
```

### タワー構成（1チームあたり）
- **ネクサス**: 1個（最終目標）
- **ネクサスタワー**: 1個（ネクサスを守護）
- **レーンインナータワー**: 3個（TOP/MID/BOT）
- **レーンアウタータワー**: 3個（TOP/MID/BOT）
- **合計**: ネクサス1個 + タワー7個

### タワーの射程

各タワーは配置されたレーンエリアのみを射程としています：

| タワー | 射程エリア |
|--------|----------|
| **TOPタワー**（アウター、インナー） | TOPレーンのみ |
| **MIDタワー**（アウター、インナー） | MIDレーンのみ |
| **BOTタワー**（アウター、インナー） | BOTレーンのみ |
| **ネクサスタワー** | ネクサスエリア（味方ネクサス付近） |

**重要**: タワーはジャングルエリアや中立エリアには攻撃できません。各レーンのタワーはそのレーンにいるミニオンのみを攻撃します。

### 進行ルート
タワーは以下の順序で破壊する必要がある：
1. **レーンアウタータワー**（各レーン）
2. **レーンインナータワー**（各レーン）
3. **ネクサスタワー**
4. **ネクサス**（破壊で勝利）

**制約**: インナータワーを攻撃するには、そのレーンのアウタータワーを先に破壊する必要がある

### エリアと位置の定義

**重要**: このゲームでは「エリア」と「位置」は同じ概念として扱います。プレイヤーは移動フェーズで以下のエリア（位置）を選択できます。

#### マップ上のエリア一覧（各チーム）

**レーンエリア**（各チームに3エリア）
1. **TOPレーン**: TOPタワーが配置されるエリア
2. **MIDレーン**: MIDタワーが配置されるエリア
3. **BOTレーン**: BOTタワーが配置されるエリア

**ジャングルエリア**（各チームに4エリア）
4. **TOP-MID森（味方陣）**: 味方側TOP-MID間のジャングル（赤バフまたは青バフ）
5. **TOP-MID森（敵陣）**: 敵側TOP-MID間のジャングル（赤バフまたは青バフ）
6. **MID-BOT森（味方陣）**: 味方側MID-BOT間のジャングル（赤バフまたは青バフ）
7. **MID-BOT森（敵陣）**: 敵側MID-BOT間のジャングル（赤バフまたは青バフ）

**中立エリア**（マップ中央、チーム共通）
8. **バロンピット**: TOP側の中立エリア、バロン（ヘラルド）が出現
9. **ドラゴンピット**: BOT側の中立エリア、ドラゴンが出現

#### エリアの隣接関係

各エリアは以下のエリアと隣接しています：

| エリア | 隣接エリア |
|--------|----------|
| TOPレーン | TOP-MID森（味方陣）、TOP-MID森（敵陣）、バロンピット |
| TOP-MID森（味方陣） | TOPレーン、MIDレーン、バロンピット |
| TOP-MID森（敵陣） | TOPレーン、MIDレーン、バロンピット |
| MIDレーン | TOP-MID森（味方陣）、TOP-MID森（敵陣）、MID-BOT森（味方陣）、MID-BOT森（敵陣） |
| MID-BOT森（味方陣） | MIDレーン、BOTレーン、ドラゴンピット |
| MID-BOT森（敵陣） | MIDレーン、BOTレーン、ドラゴンピット |
| BOTレーン | MID-BOT森（味方陣）、MID-BOT森（敵陣）、ドラゴンピット |
| バロンピット | TOPレーン、TOP-MID森（味方陣）、TOP-MID森（敵陣） |
| ドラゴンピット | BOTレーン、MID-BOT森（味方陣）、MID-BOT森（敵陣） |

#### バフ配置

**各チームのジャングルエリアに配置**:
- **赤バフ**: TOP-MID森（味方陣）またはMID-BOT森（味方陣）
- **青バフ**: TOP-MID森（味方陣）またはMID-BOT森（味方陣）
  - 通常は対角に配置（例：赤バフがTOP-MID森なら、青バフはMID-BOT森）

## ラウンド構造

### ラウンド制
ゲームは「ラウンド」単位で進行します。従来の「ターン」は「ラウンド」と呼ばれます。

**重要**: 1ラウンド = 10秒

### ラウンドの基本フェーズ

各ラウンドは以下の2つのメインフェーズで構成されます：

1. **行動宣言フェーズ**
2. **解決フェーズ**

---

### 1. 行動宣言フェーズ

各プレイヤーは同時に、秘密裏に行動を宣言します。

#### タイムリミット
- **制限時間**: 60秒
- **超過時**: 自動的に「ファーム」または「現在位置で待機」が選択される

#### 行動選択肢

**レーナーの選択肢（TOP/MID/BOT）**:
- **ファーム**:
  - ゴールド獲得（100G）
  - マッチアップ判定に不参加
  - 対面がいない場合、ミニオンラストヒットまたはタワー攻撃が可能

- **攻撃**:
  - ゴールド獲得（50G）
  - マッチアップ判定を開始
  - 対面と戦闘発生、またはミニオン/タワー攻撃

**共通の選択肢（全プレイヤー）**:
- **移動**: 他のエリアへ移動（隣接エリアのみ）
- **ガンク/ローム**: 他レーンへ移動して戦闘支援
- **オブジェクト挑戦**: ドラゴン/バロン/ヘラルド
- **ワード配置/破壊**: 視界管理
- **リコール**: 拠点に戻る（1ラウンドで完了）

**ジャングラー専用**:
- **バフファーム**: ジャングルエリアでバフを獲得（赤バフ/青バフ）

#### 宣言の同時公開

すべてのプレイヤーが宣言を完了したら、または制限時間が経過したら、すべての宣言を同時に公開します。

---

### 2. 解決フェーズ

宣言された行動を以下の順序で解決します。

#### 2-1. クリーンアップ処理
- バフ/デバフの持続時間減少
- スキルクールダウンの減少
- 期限切れバフ/デバフの削除

#### 2-2. 移動解決
- すべての移動行動を解決
- ガンク/ロームによる移動
- ワード発見・破壊判定

#### 2-3. ガンク判定（該当する場合）
- ガンク成功判定
- 視界による察知判定
- CC判定と回避判定

#### 2-4. マッチアップ発生判定
- 同じエリアにいる敵プレイヤー同士のマッチアップを判定
- 複数人戦闘の場合、ポジション設定フェーズへ

#### 2-5. ポジション設定フェーズ（複数人戦闘のみ）
- 1vs1の場合: スキップ
- ガンク成功の場合: ガンクされた側（防御側）は強制的にフロントに配置
- ファーム選択時: ファームを選択したプレイヤーが攻撃された場合、強制的にフロントに配置
- その他の複数人戦闘: 各プレイヤーがポジション（フロント/バック）を選択

#### 2-6. スキル使用宣言
- 各プレイヤーがスキル使用を宣言
- クールダウン確認

#### 2-7. マッチアップ判定
- 各ペアごとにマッチアップ判定
- ダイスロール + 攻撃力 + 各種補正

#### 2-8. ダメージ計算と適用
- ダメージ計算
- HP減少
- デス判定

#### 2-9. ファーム・リソース獲得
- ファーム選択者のゴールド獲得
- ミニオンラストヒット報酬
- バフファーム報酬

#### 2-10. オブジェクト進行
- オブジェクト挑戦の進行
- 完了判定

#### 2-11. リコール処理
- リコール中断判定
- リコール成功判定

#### 2-12. ミニオン処理
- ミニオンスポーン（3ラウンドごと）
- ミニオンのタワー攻撃
- タワーのミニオン攻撃

#### 2-13. 結果更新
- 経験値獲得
- レベルアップ判定
- 状態更新

---

### ラウンドの流れ例

**ラウンド10の例**:

1. **行動宣言フェーズ**（60秒）:
   - プレイヤーA（TOP）: 攻撃
   - プレイヤーB（JG）: ガンク（TOP）
   - プレイヤーC（MID）: ファーム
   - プレイヤーD（敵TOP）: 攻撃
   - プレイヤーE（敵JG）: バフファーム

2. **解決フェーズ**:
   - **2-1. クリーンアップ**: バフ/デバフ、スキルクールダウン更新
   - **2-2. 移動**: プレイヤーB（JG）がTOPレーンに移動
   - **2-3. ガンク判定**: プレイヤーBのガンク成功判定
   - **2-4. マッチアップ発生**: TOPレーンで2vs1（A+B vs D）
   - **2-5. ポジション設定**: プレイヤーD（ガンクされた側）は強制的にフロント配置
   - **2-6. スキル使用**: プレイヤーAがダメージスキル使用
   - **2-7. マッチアップ判定**: A vs D、B vs D
   - **2-8. ダメージ計算**: プレイヤーDがダメージを受ける
   - **2-9. リソース獲得**: プレイヤーCがファームで100G獲得
   - **2-12. ミニオン処理**: ミニオンがタワーを攻撃
   - **2-13. 結果更新**: 経験値獲得、状態更新

## 勝利条件

### 最終目標
**敵ネクサスの破壊**

### 中間目標
以下を達成することでネクサス到達が有利に：
1. タワー破壊（各レーン）
2. オブジェクト獲得による強化
3. 敵チャンピオン撃破によるリソース差
